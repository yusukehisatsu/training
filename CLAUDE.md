# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリの目的

ソフトウェアプロジェクトではなく、ユーザー個人のトレーニング・ボディメイク情報を整理する**ドキュメントリポジトリ**。

- 体の課題（肩こり・反り腰・右ハム張り・左母趾痛・浅い呼吸など）と、それに対する改善方針を整理する
- 日々のトレーニング／ストレッチメニューを管理する
- パーソナルトレーナー（Dr.トレーニング 若杉太智氏）からのフィードバックを蓄積し、メニューに反映する
- 最終的に**GitHub Pages で公開**してパーソナルトレーナーにレビューしてもらうことを想定している
- ユーザーは解剖学を学習中のため、**筋肉名・関節・動きのメカニズムを科学的なレベル**で記述する方針

## ディレクトリ構成

```
docs/       GitHub Pages 公開ディレクトリ（GitHub Pages 設定の Folder: /docs）
  index.html              トップページ
  assets/
    css/style.css         スタイル
    js/app.js             ハッシュルーター＋Markdown 描画
  content/                公開されるデータソース（HTML/CSS/JS が読み込む Markdown）
    issues.md             現在の課題と解決の方向性
    menu/
      gym.md              ジムメニュー
      home.md             自宅トレーニングメニュー
      stretch.md          ストレッチメニュー
      lifestyle.md        生活習慣（食事・呼吸法・姿勢）
resource/   外部から取り込んだ参考情報（一次ソース。公開対象外）
  chatgpt.md              身体課題と改善方針の全体像（マスタードキュメント）
  kachidok_machine.md     ジム（Blue Fitness 24 勝どき店）のマシン一覧
  kachidok_machine.pdf    マシン一覧の元 PDF
  trainer/
    feedback/             トレーナーから受けた回答を YYYYMMDD.md で保存（日付不明分は 2025xxxx.md などの形）
    proposed_menu/        トレーナーから提案されたメニュー集（やり方の原典）
      menu_list.md
```

`docs/content/` 配下の Markdown は GitHub Pages 上で `docs/assets/js/app.js`（marked.js を使うハッシュルーター）から fetch されて描画される「データソース」として扱う。ナビゲーションリンクや表示用の装飾は入れず、内容そのものに集中する。

`docs/` を GitHub Pages の公開単位とし、それ以外（`resource/`・`CLAUDE.md`・`README.md`）は内部資料として公開対象外。

### ローカル確認

`docs/assets/js/app.js` は `fetch()` で Markdown を読むため `file://` では動かない。確認時は：

```sh
cd docs
python3 -m http.server 8080
# → http://localhost:8080/
```

## 編集時のルール

- **トレーナーフィードバックを反映するときは、元のフィードバックファイルは原文のまま残す**こと。反映先は `docs/content/` 配下を更新する
- 新しいメニューを追加する際は `resource/trainer/proposed_menu/menu_list.md` のフォーマット（`- [ ] メニュー名` + `やり方` のネスト箇条書き）に揃える
- 解剖学的な筋肉名（例：僧帽筋下部、肩甲下筋、腹横筋、内腹斜筋）は略称ではなく**正式名称**で書く
- メニュー説明では、**意識すべき部位／避けるべきフォーム**（例：「肩をすくめない」「腰を反らない」「肩甲骨を下制」）を必ず記載する。これがユーザーにとってのコア情報
- トレーナーから「外すべき」と指摘された種目（例：ロータリートルソー）は、削除するのではなく**理由つきで非推奨マーク**を残すと、後から経緯を追える
- **`docs/content/` 配下の Markdown は他ファイルへのパス参照（`docs/content/...` や `resource/...` 等）を含めない**。各ファイルは自己完結させる方針（クロスリンクは過去に意図的に削除済み）

## 内容上の重要な前提（書き換えや提案を行う際に踏まえる）

### 身体的な前提
- 中心仮説：肩・首・腰・ハム・足指の不調は別々ではなく、**呼吸・肋骨・肩甲骨・骨盤・股関節・足部の連動不全**としてつながっている
- 優先順位（`resource/chatgpt.md` 参照）：呼吸 → 肩甲骨下制 → 反らずに胸を張る → 腹斜筋／体幹活性 → 股関節・臀筋 → 足部 → フォーム優先のウェイト
- トレーナーによる**活性すべき筋／抑制すべき筋の分類**は `resource/trainer/feedback/20260315.md` にある。新しいメニュー提案時はこの分類と矛盾していないか確認する
- 左母趾の状況により、有酸素は**衝撃の少ない種目（リカンベントバイク・エリプティカル・スピニングバイク）**が望ましい

### ジムメニュー設計の制約（`docs/content/menu/gym.md`）
- 朝イチ出勤前のジム、**滞在 25 分、基本毎日**（土曜のみ完全休）
- 利用ジム：Blue Fitness 24 勝どき店（MATRIX／SPIRIT／DRAX／ZERO-i）
- **抑制すべき筋を直接狙う種目はメニューに組まない**（押す日＝胸・肩は設けない、リアデルト・バイセプスカール等もカット）
- 重視ポイント：①背中（広背筋・僧帽筋下部） ②脚（デッドリフト中心、ハム・大臀筋・中臀筋） ③有酸素1日
- モビリティ（ZERO-i 等）は毎朝独立してやる前提のため、ジムメニューには含めない

## GitHub Pages 公開を見据えた書き方

- 将来的にトレーナーが Web 上でレビューする前提なので、Markdown は**素の Markdown として読みやすい**形を保つ（独自記法や前提知識を要する省略を避ける）
- 動画リンクは `[テキスト](URL)` の形式で残す（`menu_list.md` 既存パターン）
- `app.js` には `LINK_MAP` で旧来の `xxx.md` 参照を内部ナビゲーションへ書き換える後方互換コードがあるが、**新規には書かない**方針
